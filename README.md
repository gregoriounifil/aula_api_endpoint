GET /api/produtos -> Lista os produtos com filtro. <br>
GET /api/produto/:id -> Busca um produto pelo seu id. <br>
POST /api/produtos -> Cadastra um produto. <br>
POST /api/categorias -> Cadastra uma categoria. <br>
POST /api/usuarios -> Cadastra um novo usuário. <br>
POST /api/vendas -> Registra a venda de um produto. <br>
<br>
Os tipos e as validações dos campos dentro dos métodos POST e GET da API garantem a integridade dos dados que serão inseridos. Isso permite, também, aplicar as regras de negócio, tornando inválidas certas informações, o que evita erros de processamento e resulta em uma API completa e funcional. Além disso, o sistema valida e confirma a existência de IDs criados por meio de UUID, o que confere uma funcionalidade profissional à API e assegura a correta comunicação entre tabelas.
